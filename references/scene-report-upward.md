# Scene: Upward Report

适用于：

- 向上级汇报工作
- 阶段情况报告
- 汇报问题、思路、打算

## 这个场景要解决什么

向上汇报不是大会动员，也不是内部讨论。它最看重四点：

1. 情况是否清楚
2. 判断是否准确
3. 问题是否讲透
4. 思路是否稳妥

它不能写得虚，也不能写得炸。关键是准。

## 推荐骨架

### 1. 先交代基本态势

先用几句话说清：

- 当前总体情况怎样
- 处于什么阶段
- 主流是什么

### 2. 再讲最关键的成绩和变化

不是全面铺开，而是抓能说明趋势和基础的内容。

### 3. 问题部分要分层

向上汇报时，问题不能写成一锅粥。至少分清：

- 哪些是当前最突出的
- 哪些是阶段性问题
- 哪些是深层制约

### 4. 原因分析要克制但要到位

尽量分清：

- 是认识不到位
- 是推进不够实
- 是机制不顺
- 还是条件不足

不要一概归咎，也不要一句“原因复杂”带过。

### 5. 最后汇报思路和建议

向上汇报最关键的，不只是报情况，而是要说明：

- 下一步准备抓什么
- 为什么先抓这个
- 需要哪些支持或协调
- 风险点在哪里

## 语气要求

- 客观
- 稳健
- 不回避问题
- 不故作惊人
- 不空喊态度

## 一个简版模板

1. 当前态势
2. 主要进展
3. 突出问题
4. 原因判断
5. 下一步思路和建议
