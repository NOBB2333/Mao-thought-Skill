# Scene: Issue Analysis

适用于：

- 平时问题分析
- 内部研讨
- 专题探讨
- 情况诊断和思路碰撞

## 这个场景要解决什么

这类文本最重要的，不是立刻形成正式结论，而是把问题看透一点、分清一点、往前推进一点。

它要避免两个极端：

- 只有材料，没有判断
- 只有态度，没有分析

## 推荐骨架

### 1. 先把问题讲准

一句话说明：

- 我们现在到底在讨论什么
- 这个问题具体表现在哪

### 2. 把现象和本质分开

不要被表面现象带着走。先区分：

- 看得见的表现是什么
- 背后真正起作用的因素是什么

### 3. 抓主要矛盾

问题往往不止一个，但讨论时必须逼近那个最带全局的点。

要问：

- 如果只先解决一件事，最该解决什么
- 这个问题为什么总在别的问题后面反复冒出来

### 4. 分清性质

平时探讨尤其要防止乱上纲。尽量分清：

- 是局部现象还是普遍倾向
- 是执行偏差还是制度缺陷
- 是认识差异还是利益冲突
- 是短期卡点还是长期积累

### 5. 提出可讨论的方向

不要把结尾写成空泛正确。至少给出：

- 一个总判断
- 两三个可能抓手
- 各自的利弊或前提

## 语气要求

- 清楚
- 朴素
- 有分析力
- 可以保留不确定性
- 不要为了“显得深刻”故意写晦涩

## 一个简版模板

1. 问题是什么
2. 主要现象
3. 主要矛盾
4. 性质判断
5. 几种思路或抓手
